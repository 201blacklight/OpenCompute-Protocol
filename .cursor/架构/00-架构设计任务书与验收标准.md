# 架构设计任务书与验收标准

## 1. 架构设计 Agent 集群职责
- `Solution-Architect-Agent`：总体架构和边界定义。
- `Protocol-Agent`：协议字段、状态机、版本兼容设计。
- `Security-Agent`：鉴权、签名、密钥管理、风控设计。
- `Data-Agent`：账本模型、对账模型、审计日志模型。
- `Reliability-Agent`：熔断、重试、降级、监控告警。

## 2. 必须提交的架构文档
- `system-context.md`：上下游系统边界图（文本版）。
- `protocol-spec.md`：消息格式、签名规则、错误码。
- `billing-ledger.md`：Token发行、消耗、冻结、回滚规则。
- `security-compliance.md`：威胁模型与合规映射。
- `runbook.md`：故障与回滚手册。

## 3. 决策层审计清单（通过标准）
- 数据一致性：同一订单在所有子账本可对齐。
- 资金安全：不托管资金，回执驱动记账，不越权调用。
- 合规可解释：每个核心流程有审计事件ID。
- 成本可控：单次请求链路复杂度可量化。
- 可演进：协议版本支持向后兼容。

## 4. 非通过即打回条件
- 出现单点账本或不可回滚资金状态。
- 缺少防重放签名和幂等键。
- 缺少审计日志或日志不可关联订单ID。
- 缺少异常状态处理（超时、重复回执、部分失败）。
