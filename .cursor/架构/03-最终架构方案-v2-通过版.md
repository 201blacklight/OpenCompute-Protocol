# OCP 最终架构方案 v2（审计通过）

## 1. 目标
- 提供统一协议完成“支付回执驱动Token额度发放、模型调用扣费、分成记账、可审计追踪”。
- 在不托管资金前提下，完成可演示、可复现、可扩展的基础设施。

## 2. 组件清单（通过版）
- `Gateway + Auth`
  - OAuth2/OIDC接入
  - 请求签名验证和速率限制
- `Receipt Ingestion`
  - 支付回执校验、验签、防重放
  - 幂等处理（idempotency key）
- `Token Engine`
  - 双轨汇率（基础汇率 + 动态系数）
  - 发放、冻结、解冻、冲正
- `Usage Metering`
  - 模型调用计量、SLA标签、费用计算
- `Settlement & Share`
  - 分账规则执行
  - 收益分成权登记与转移（含争议冻结）
- `Ledger`
  - 双分录账本（借贷平衡）
  - 事件溯源日志（append-only）
- `Audit & Policy`
  - 审计报表、试点报表、异常告警
- `Adapter SDK`
  - 支付适配器SDK
  - 模型厂商适配器SDK

## 3. Ledger 状态机（关键）
- `INIT` -> `PENDING_RECEIPT`
- `PENDING_RECEIPT` -> `CREDITED` | `REJECTED`
- `CREDITED` -> `RESERVED` -> `CONSUMED`
- `RESERVED` -> `RELEASED`（解冻）
- `CONSUMED` -> `REFUND_PENDING` -> `REFUNDED`
- 任意状态 -> `DISPUTED`（争议冻结）
- `DISPUTED` -> `RESOLVED`（裁定后回到对应状态）

## 4. 安全与风控
- 每次回执必须包含：渠道签名、时间戳、nonce。
- 回执写入前先查幂等键，重复请求直接返回历史结果。
- 扣费使用乐观锁版本号，避免并发超扣。
- 异常检测：
  - 汇率异常波动
  - 调用量突增
  - 单账户分成异常

## 5. 政府和大厂支持策略（落到架构）
- 提供监管视图API（按地区、行业、模型类型聚合）。
- 提供企业接入SDK与沙箱环境，3步完成最小接入。
- 提供审计追溯链路（trace id -> order id -> ledger entries）。

- 风险可控度：通过（在“非托管资金”边界内）
- Demo可实现性：通过
