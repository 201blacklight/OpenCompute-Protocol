# OCP 架构方案 v1（首提）

## 1. 系统分层
- 接入层：`API Gateway`、`Auth Service`
- 协议层：`Token Exchange Service`、`Pricing Engine`
- 交易层：`Payment Receipt Service`、`Settlement Service`
- 资产层：`Usage Ledger`、`Revenue Share Registry`
- 适配层：`Payment Adapter`、`Model Provider Adapter`
- 治理层：`Audit Service`、`Risk Engine`

## 2. 核心流程（v1）
1. 用户发起充值（外部支付）。
2. 支付渠道回执进入 `Payment Receipt Service`。
3. `Token Exchange Service` 根据汇率发放Token额度。
4. 用户调用AI模型，`Settlement Service`扣减Token。
5. 收入按规则写入 `Revenue Share Registry`。

## 3. 协议对象（简版）
- `Account`: 用户、商户、厂商主体。
- `TokenBalance`: 可用、冻结、已消耗。
- `PaymentReceipt`: 渠道回执与签名。
- `UsageRecord`: 模型调用记录和费用。
- `SettlementRecord`: 分账结果和状态。

## 4. v1 未解决问题
- 汇率缺少防操纵机制。
- 对账规则未细化到“部分成功”场景。
- 收益分成权转移流程缺少争议处理。
