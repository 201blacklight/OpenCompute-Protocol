# 执行层 Agent 分工与监督机制

## 1. 执行层分工
- `Backend-Agent`：实现协议核心逻辑（充值、扣费、分成、转移）。
- `Adapter-Agent`：实现支付回执和模型调用的模拟适配器。
- `QA-Agent`：编写测试用例和回归脚本。
- `Ops-Agent`：运行脚本、演示脚本、交付打包。
- `PM-Agent`：跟踪任务燃尽和验收状态。

## 2. 交付节奏
- D1：完成数据模型和状态机。
- D2：完成核心服务与命令行Demo。
- D3：完成测试和文档，提交决策层验收。

## 3. 监督要求
- 每日同步：新增风险、阻塞项、预计完成时间。
- 每次提交必须附带：
  - 变更说明
  - 测试结果
  - 回滚方法
- 未通过项必须24小时内给出修复方案。
