# OpenCompute Protocol（OCP）项目北极星与边界

## 1. 北极星目标
- 用统一协议连接多AI厂商的计费与授权体系，降低用户在不同AI之间切换成本。
- 在合法合规前提下，把“现实货币 -> AI可消费额度”标准化，形成可审计的结算层。
- 先做“支付与配额互通”，再逐步探索“算力资产化与所有权交易”。

## 2. 现实约束（无资金、无人脉）
- 不做重资产路线，不依赖自建大规模算力。
- 首阶段不触碰资金池和托管，不碰跨境清结算牌照红线。
- 通过开源标准、可验证Demo、监管友好文档，换取政府试点和大厂合作机会。

## 3. 分阶段战略
- P0（0-3个月）：定义协议草案、完成离线模拟Demo、完成风险白皮书。
- P1（3-9个月）：接入1-2个支付聚合服务沙箱 + 1-2个模型供应方沙箱，跑通闭环。
- P2（9-18个月）：政产学试点，形成行业联盟版本（标准 + 审计 + SLA）。

## 4. 原始想法落地改造（关键取舍）
- 保留：统一结算单位（Token币）和跨AI消费体验。
- 延后：AI所有权买卖（先做“收益分成权”而非“完整所有权转让”）。
- 延后：总量锚定“全网极限算力”这种难以实时验证的机制，改为分层锚定：
  - 协议层总量（治理可调，审计可追溯）
  - 厂商层额度（厂商声明 + 可验证消耗日志）
  - 用户层余额（账户可查、可冻结、可回滚）

## 5. 合规与合作定位
- 政府价值：数字经济基础设施、AI消费透明化、反欺诈与审计能力。
- 大厂价值：新增分发渠道、统一计费接入、降低获客与结算成本。
- 用户价值：多AI统一余额、统一账单、价格可比、权益可迁移（在协议允许范围）。
