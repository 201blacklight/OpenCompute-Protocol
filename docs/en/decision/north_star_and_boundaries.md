# OCP North Star and Boundaries

## North Star
- Build a unified protocol for AI payment, quota, and settlement interoperability.
- Convert user purchasing power into protocol token credits under auditable rules.
- Start with practical interoperability, then expand to broader compute finance primitives.

## Real Constraints
- No initial capital and no existing partner network.
- Must avoid regulated fund custody and cross-border clearing red lines.
- Must prove value through open standards and reproducible demos first.

## Strategic Sequencing
- Phase 0: protocol draft + offline demo + risk whitepaper.
- Phase 1: sandbox integrations (payment + model provider).
- Phase 2: government/industry pilot and open alliance standardization.

## Key Scope Decisions
- Keep: unified tokenized usage settlement.
- Delay: full AI ownership trading; start with revenue-share rights transfer.
- Replace global hard-cap assumption with layered caps:
  - protocol cap
  - provider cap
  - user cap
