# 故障处理手册

## 场景A：接口 401/403

### 现象
- 返回 `HTTP_401` 或 `HTTP_403`

### 排查
1) 检查 `X-API-Key` 是否存在且有效
2) 检查角色是否有对应接口权限
3) 核对调用路径是否正确（例如审计接口仅 auditor/admin 可用）

### 处理
- 修正 API Key 或调整角色权限映射

---

## 场景B：Quote 偶发超时

### 现象
- Provider 调用超时（尤其 `anthropic`）

### 排查
1) 查看日志是否有重试痕迹
2) 统计超时占比是否超过阈值

### 处理
- 默认已重试 3 次；如仍失败，切换 `providerId=mock` 保持业务可用

---

## 场景C：Consume/Reconcile 重复提交

### 现象
- 上游重复调用导致潜在重复记账风险

### 排查
1) 检查是否带 `Idempotency-Key`
2) 确认日志是否命中 `idempotent_hit`

### 处理
- 要求上游必须传递稳定幂等键
- 对未传幂等键的调用增加网关限制

---

## 场景D：Reconcile 大量 MISMATCH

### 现象
- 对账结果频繁 `MISMATCH`

### 排查
1) 检查 provider 对账摘要输入是否正确
2) 核对统计周期是否一致（`periodStart/periodEnd`）
3) 核查 usage 数据是否存在延迟写入

### 处理
- 先出具临时差异报表，启动人工复核
- 对同批次数据执行补算并二次对账
